# Дипломный проект "Разработка веб-сервиса для чтения графической литературы"
Я начал разработку дипломной работы бакалавра. Моя идея - веб-сервис для чтения манги под названием "MangeDex". Главной фишкой сервиса будет продвинутая система персональных рекомендаций (сервис должен будет анализировать как предпочитаемый пользователем контент, так и паттерны его поведения, сопоставлять параметры с пользователями, похожими на него, чтобы предложить максимально релевантный контент, рассчитав, при этом, процент совместимости (то есть, система, похожая на алгоритмы рекомендаций Netflix, Amazon и Spotify)).

## Технологический стек
Бекенд будет написан на Java 25 и Spring Boot 4. Данные храниться в PostgreSQL и медиа в MinIO. Системой аутентификации будет Keycloak. Все сервисы будут представлять собой Docker-контейнеры.

## Архитектура
Архитектура приложения будет представлять собой модульный монолит.

## Функционал
Сервис должен предоставлять:

1. Персональные коллекции пользователей ("Никогда не забуду", "Популярно в Тик-Токе" и т.д.) с возможностью делиться по ссылке (как это можно делать с плейлистами в музыкальных сервисах) и настройкой приватности
2. Просмотр всех глав произведения
3. Поиск по фильтрам (название, теги, жанры, количество глав, возвратной рейтинг, тип, формат выпуска, статус выпуска тайтла/перевода) и сортировка результатов (по рейтингу, по релевантности, по названию, по просмотрам, по количеству глав, по дате релиза, по дате обновления/добавления и т.д.)
4. Списки подборок на главной странице (например, популярно сейчас, последние обновления и т.д.)
5. Страницы тайтлов с описанием, обложкой, списком глав, рейтингом, тегами, жанрами, информацией о статусе выпуска и переводе
6. Систему рейтингов тайтлов (оценка в лайках/дислайках)
7. Систему рекомендаций на основе предпочтений и поведения пользователей
8. Систему аутентификации и авторизации пользователей
9. История чтения пользователя
10. Панель администратора для управления контентом
